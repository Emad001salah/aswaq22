# 🚀 Aswaq Platform: Master Scalability Roadmap

هذا الدليل يمثل المرجعية التقنية لتحويل منصة "أسواق" من نموذج أولي (MVP) إلى منصة عالمية قادرة على استيعاب ملايين المستخدمين المتزامنين في 22 دولة عبر 3 قارات.

---

## 🏗️ 1. المبدأ المعماري: "توسع بنقرة زر"
تم بناء الهيكل الحالي للمشروع باستخدام (Next.js + Prisma) لضمان سهولة الانتقال بين الأنظمة دون الحاجة لإعادة كتابة الكود من الصفر.

### الخطوات الاستراتيجية:
- **نظام البيانات**: الانتقال من SQLite المحلي إلى **Managed PostgreSQL (Google Cloud SQL)**.
- **توزيع الحمل (Sharding)**: سيتم تقسيم الاستعلامات برمجياً بناءً على `countryId` لضمان عدم حدوث اختناقات في قواعد البيانات.
- **القراءة المتوازنة**: تفعيل **Read Replicas** لتوزيع ضغط التصفح الهائل بعيداً عن عمليات الكتابة (البيع/الشراء).

---

## 🔍 2. محرك البحث والذكاء اللحظي
البحث التقليدي لن يصمد أمام ملايين الإعلانات، لذا سنعتمد على:
- **Algolia / Elasticsearch**: للبحث النصي الفوري والبحث الجغرافي (Geo-search) بنتائج تظهر في أقل من 100ms.
- **Cache-First Design**: استخدام طبقة كاش وسيطة تدعم **Redis** لإراحة قاعدة البيانات من الاستعلامات المتكررة بنسبة 80%.
- **CDN (Content Delivery Network)**: توزيع الصور والفيديوهات عبر شبكات Google العالمية لضمان سرعة التحميل في أي بلد.

---

## ⚡ 3. استراتيجية الكاش (Caching Strategy)
لضمان الأداء الأقصى، نتبع القواعد التالية:
- **البيانات الثابتة**: كاش طويل الأجل (1 ساعة) للتصنيفات والإعدادات العامة.
- **البحث والتصفح**: كاش قصير الأجل (1 دقيقة) لنتائج البحث مع تفعيل **Cache Invalidation** فوري عند إضافة محتوى جديد.
- **البيانات الحساسة**: المدفوعات والاشتراكات لا يتم عمل كاش لها نهائياً (Real-time only).

---

## 🛡️ 3. الحماية والمرونة (Resilience & Security)
- **Multi-Layer Rate Limiting**: تطبيق حماية مزدوجة تعتمد على الـ IP والـ User ID معاً لمنع تلاعب الحسابات الفردية أو الهجمات الموزعة.
- **Burst Protection**: حماية من النبضات المفاجئة (Sudden Spikes) تكتشف الهجمات اللحظية وتوقفها في أقل من 2 ثانية، مما يحمي قاعدة البيانات من الانهيار.
- **Behavioral Security**: نظام ذكي يحلل وتيرة الطلبات ويمنع الـ Bots والـ Scrapers قبل وصولهم لمنطق العمل (Business Logic).
- **Cloud Run Auto Scaling**: تفعيل التوسع التلقائي للسيرفرات بناءً على حجم الحركة اللحظية.

---

## 🏗️ 4. الجاهزية الميدانية (Field Readiness Checklist)
للانتقال من التطوير إلى العمل الحقيقي في الأسواق، يجب تفعيل هذه الميزات:
- **نظام التحقق (KYC Lite)**: توثيق حسابات البائعين عبر رقم الهاتف لزيادة الموثوقية.
- **نظام الرقابة (Moderation Queue)**: مراجعة الإعلانات قبل نشرها للتأكد من جودتها ومنع الاحتيال.
- **إدارة دورة حياة الإعلان**: أتمتة انتهاء صلاحية الإعلانات (30 يوم مثلاً) وتنبيه البائع للتجديد.
- **نظام التقييم (Trust & Reviews)**: السماح للمشترين بتقييم تجربتهم مع البائع لبناء "سمعة برمجية".

---

## 💬 5. المراسلة والاتصال اللحظي
- **Firebase Realtime Database**: للدردشة اللحظية بين البائع والمشتري وتتبع الشحنات، لضمان استهلاك منخفض جداً لموارد الخادم الرئيسي ودقة عالية جداً.

---

## 🛠️ 5. حالة المشروع الحالية (Audit)
| الميزة | الحالة الحالية (MVP) | الحالة المستهدفة (Scaling) |
| :--- | :--- | :--- |
| **قاعدة البيانات** | Prisma + SQLite (ملف محلي) | Prisma + PostgreSQL (Cloud SQL) |
| **البحث** | Prisma Database Queries | Algolia / Elasticsearch |
| **الملفات** | Local Storage / Static Assets | CDN + GCS (Google Cloud Storage) |
| **الحماية** | Basic Node Middleware | Cloud Armor + API Rate Limiting |

---

## ⚡ 6. دليل التحول السريع (One-Click Migration Guide)
عندما تصل المنصة إلى **100k+** مستخدم نشط، اتبع هذه الخطوات للتحول السحابي:

### أولاً: قاعدة البيانات (Database)
1. قم بإنشاء قاعدة بيانات **Google Cloud SQL (PostgreSQL)**.
2. حدث ملف `.env` برابط القاعدة الجديدة.
3. غير الـ `provider` في `schema.prisma` إلى `postgresql`.
4. نفذ `npx prisma db push` لنقل الهيكل (Schema) فوراً.

### ثانياً: التخزين المؤقت (Redis)
1. قم بتفعيل **Google Cloud Memorystore (Redis)**.
2. في ملف `scalability.ts` قم باستبدال الـ `Map` بـ `ioredis`.
3. لا تقم بتغيير أي كود في الـ API، وسيبدأ النظام في سحب البيانات من Redis تلقائياً.

### ثالثاً: الصور والملفات (CDN)
1. اربط `uploads/` بـ **Google Cloud Storage (GCS)**.
2. فعل **Cloud CDN** لضمان وصول الصور للمستخدمين في اليمن، الأردن، والسعودية بأقل من 50ms.

---

> **نصيحة تقنية**: المنصة الآن ليست مجرد كود، بل هي **Micro-Architecture** جاهزة للنمو اللحظي. كل ما تحتاجه هو "تبديل المحركات" فقط.
