# 📱 خطة التحويل الكاملة لتطبيق الهاتف الأصلي | Aswaq Native Mobile Roadmap
> **دعم متجري Google Play و Apple App Store باستخدام Capacitor**

تحتوي هذه الوثيقة الشاملة على **الخطة التقنية التفصيلية** و**ملفات التكوين الجاهزة** التي قمنا بإعدادها وتنزيلها للتو داخل المشروع، لتحويل تطبيق الويب التفاعلي **أَسْوَاق (Aswaq)** إلى تطبيق هاتف ذكي أصلي (Native Mobile App) فائق الأداء والسرعة.

---

## 🛠️ القسم 1: لماذا Capacitor هو الاختيار الأمثل لمشروع "أسواق"؟
عند تحويل تطبيق ويب مبني بـ React و Tailwind و WebSockets وخريطة تفاعلية وقاعدة بيانات، هناك خياران رئيسيان:
1. **React Native**: يتطلب إعادة كتابة واجهات المستخدم (UI) والاتصالات بالكامل من الصفر بـ JSX خاص بالهواتف، مما يعني مضاعفة وقت التطوير ومشاكل التزامن.
2. **Capacitor (المعتمد حالياً)**: يحزم كود الويب عالي الأداء داخل حاوية WebView native فائقة السرعة مع توفير جسور اتصال برمجية (JS-to-Native Bridges) للوصول لكامل حساسات الهاتف (الكاميرا، نظام الـ GPS، البصمة، الإشعارات الفورية).

### مميزات Capacitor لمشروع أسواق:
* **كود موحد تماماً (Single Codebase)**: أي تعديل تطلقه في واجهة الويب ينعكس فوراً وتلقائياً على تطبيق الأندرويد والآيفون.
* **الأداء العالي**: توافق تام مع محركات الويب الأصيلة للموبايل (WebKit في iOS و Chromium في Android).
* **الدعم الفني واللوائح**: معترف به رسمياً بواسطة Apple و Google ومستخدم في أكبر التطبيقات العالمية.

## 📂 القسم 2: ملفات التكوين والتهيئة التي تم إعدادها بالفرع الرئيسي
لقد قمنا بتهيئة وتثبيت البيئة الأساسية بالمشروع كالتالي:

1. **تكوين Capacitor الرئيسي (`capacitor.config.json`)**:
   * تحديد معرّف التطبيق الفريد: `com.aswaq.app`.
   * توجيه الحزمة لبناء المخرجات داخل مجلد `dist/`.
   * إعداد تراكيب الحماية ونظام التصفح الخارجي الآمن لموارد الخرائط.
   * إعداد واجهات شاشات التمهيد والتحميل (Splash Screen) وشريط الحالة (Status Bar) لتتوافق مع الهوية البصرية الغامقة للمنصة.

2. **تكامل شريط الحالة الذكي (`src/context/ThemeContext.tsx`)**:
   * تمت إضافة بروتوكول يغير مظهر شريط حالة الهاتف تلقائياً إلى النمط الداكن أو الفاتح فور تبديل المستخدم للـ Theme بالموقع الإلكتروني.

3. **جسر الاتصالات البرمي الأصلي (`src/lib/native.ts`)**:
   * أنشأنا بروتوكولاً ذكياً يقوم تلقائياً بطلب الصلاحيات الحساسة من خلال حساسات الهاتف الحقيقية (Camera, Geolocation Native System, Native Share Sheet) مع توفير خط رجعة ذكي وآمن للعمل على المتصفحات القياسية إذا كان المستخدم يتصفح الويب.

## 🚀 القسم 3: خطوات بناء وتشغيل التطبيق الآن

اتبع الخطوات التالية بحسب نظام التشغيل المستهدف:

### 1️⃣ الخطوة الأولى: بناء نسخة الويب المستقرة
قم بتجميع الكود واختباره بالكامل:
```bash
npm run build
```

### 2️⃣ الخطوة الثانية: إضافة المنصات (Android & iOS)
لتوليد كود مشاريع أندرويد ستوديو وإكس كود الأصلية:
```bash
# لإضافة مشغل الأندرويد
npm run cap:add-android

# لإضافة مشغل الآيفون
npm run cap:add-ios
```

### 3️⃣ الخطوة الثالثة: مزامنة المجلدات والموارد البرمجية
عند إجراء أي تغيير في كود الواجهة، قم ببناء الويب ومزامنتها بضغطة واحدة:
```bash
npm run build
npm run cap:sync
```

### 4️⃣ الخطوة الرابعة: تشغيل التطبيق في بيئة المحاكاة الرسمية
سيقوم هذا الأمر بفتح البيئة ومحاكي الأجهزة الحقيقي فوراً:
```bash
# لفتح بيئة Android Studio وتحميل المحاكي
npm run cap:open-android

# لفتح بيئة Xcode وتحميل المحاكي الخاص بـ iPhone
npm run cap:open-ios
```

## 🎨 القسم 4: توليد الأيقونات وشاشات التمهيد تلقائياً

لتحديث أيقونة التطبيق وشاشات التمهيد (Splash Screen) بضغطة زر واحدة تناسب مقاسات iOS و Android الكاملة:

1. قم بتثبيت أداة توليد الأصول:
```bash
npm install -g @capacitor/assets
```
2. ضع صور الأيقونات والتمهيد داخل مجلد مخصص بالمشروع:
   * أيقونة التطبيق: بمقاس `1024x1024` بيكسل باسم `assets/icon.png`.
   * شاشة التحميل: بمقاس `2732x2732` بيكسل باسم `assets/splash.png`.
3. قم بتشغيل الأمر السحري التالي:
```bash
npx capacitor-assets generate
```
سيقوم المولد بقص وضبط واحتواء جميع الصيغ والمقاسات للهواتف تلقائياً ودمجها بداخل مجلدات الأندرويد والآيفون.

## 🔐 القسم 5: تهيئة وتأكيد أذونات الكاميرا والـ GPS بالمنصات

يجب تضمين مبررات واضحة لطلب الصلاحيات حتى يُقبل التطبيق بالمتجر:

### 📱 نظام iOS (تعديل ملف `Info.plist` في Xcode):
افتح `ios/App/App/Info.plist` وأضف الأسطر التالية لتقديم سبب مقنع للمراجعين بمتجر آبل:
```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>يطلب تطبيق أسواق صلاحية تتبع موقعك الجغرافي لعرض الإعلانات التجارية الأقرب إليك ولتتبع خط سير المندوب بالشحنات.</string>
<key>NSCameraUsageDescription</key>
<string>يطلب تطبيق أسواق الوصول للكاميرا لالتقاط صور السلع والمنتجات وإرفاقها فورياً بالإعلان الخاص بك.</string>
<key>NSPhotoLibraryAddUsageDescription</key>
<string>يطلب تطبيق أسواق صلاحية حفظ صور الفواتير والعروض والدردشة بداخل ألبوم الصور الخاص بك.</string>
```

### 🤖 نظام Android (تعديل ملف `AndroidManifest.xml`):
افتح `android/app/src/main/AndroidManifest.xml` وتأكد من تضمين الصلاحيات التالية بمستوى الجذر:
```xml
<!-- صلاحية تحديد الموقع بالـ GPS والشبكات -->
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />

<!-- صلاحية الوصول واستعمال الكاميرا لالتقاط سلع أسواق -->
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
```

## 🌟 القسم 6: خطوات الرفع والنشر النهائي لـ Google Play و App Store

### 📦 متجر جوجل بلاي (Google Play Console):
1. **أندرويد ستوديو**: افتح المشروع بـ `npm run cap:open-android`.
2. **صناعة نسخة التوقيع (Signing Bundle)**: من الشريط العلوي حدد `Build` > `Generate Signed Bundle / APK`.
3. **التضمين الجغرافي**: اختر `Android App Bundle (.aab)` لتوفير مساحة تحميل فائقة للمستخدمين.
4. **التوقيع الرقمي**: أنشئ مفتاح حماية وتوقيع رقمي مسجل خاص بشركتكم.
5. **الرفع للمتجر**: ارفع ملف الـ `.aab` المنتج لمتجر كونسول وابدأ الاختبار المغلق ثم النشر الكامل للمستخدمين.

### 🍏 متجر آبل (Apple App Store Connect):
1. **اشتراك المطورين**: تأكد من امتلاك حساب Apple Developer مدفوع.
2. **بيئة Xcode**: افتح المشروع بـ `npm run cap:open-ios`.
3. **تحديد الهوية والشهادة**: اختر جهاز المحاكاة كـ `Any iOS Device` وحدد التوقيع الآلي `Signing & Capabilities` > `Automatically manage signing`.
4. **الأرشفة (Archive)**: اختر من الأعلى `Product` > `Archive`.
5. **إرسال النسخة (Distribute)**: اضغط على زر `Distribute App` وحدد خيار الإرسال لمنصة `App Store Connect`.
6. **التحقق الخارجي**: قم بإكمال لوحة تفاصيل التطبيق واشتراطات الخصوصية داخل موقع App Store Connect واضغط على زر "إرسال للمراجعة النهائية".
